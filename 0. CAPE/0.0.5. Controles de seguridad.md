 
Después de ganar un punto de apoyo, podríamos utilizar este acceso para tener una pista sobre el estado defensivo de los anfitriones, enumerar el dominio más ahora que nuestra visibilidad no está tan restringida, y, si es necesario, trabajar en "vivir fuera de la tierra" usando herramientas que existen nativamente en los anfitriones.

## Windows Defender


Windows Defender (o [Microsoft Defender](https://en.wikipedia.org/wiki/Microsoft_Defender) después de la actualización Windows 10 de mayo de 2020) ha mejorado mucho con los años y, de forma predeterminada, bloqueará herramientas como `PowerView`. Hay maneras de eludir estas protecciones. Estas maneras se abordarán en otros módulos. Podemos utilizar el PowerShell cmdlet [incorporado Get-MpComputerStatus](https://docs.microsoft.com/en-us/powershell/module/defender/get-mpcomputerstatus?view=win10-ps) para obtener el estatus actual de Defender. Aquí, podemos ver que el `RealTimeProtectionEnabled`parámetro está listo para `True`, lo que significa que el Defensor está habilitado en el sistema.


#### Comprobación del Estado del Defender

```PowerShell
Get-MpComputerStatus
```

## AMSI

AMSI (AntiMalware Scan Interface), es una API que permite que las aplicaciones y servicios se integren con cualquier producto antimalware que esté presente en la máquina. 
• AMSI está diseñada para permitir las técnicas más comunes de escaneo y protección proporcionadas por los productos antimalware. Permite el escaneo de archivos, memoria o flujos, la comprobación de la reputación de la URL/IP de origen y otras técnicas. 
Componentes de Windows que se integran con AMSI: 
o UAC (elevación de EXE, COM, MSI o instalación de ActiveX) 
o PowerShell (scripts, uso interactivo y evaluación dinámica de código) 
o Windows Script Host (wscript.exe y cscript.exe) 
o JavaScript, VBScript y VBA (Macros Office) 
Cuando un script está listo para ser suministrado al intérprete (p.ej. un .ps1 a PowerShell), se puede invocar a AMSI para solicitar al antimalware un análisis del contenido antes de ejecutarlo. Esto ocurre incluso si el script se genera en tiempo de ejecución: el código puede pasar por una desofuscación pero, en última instancia, debe proporcionar al intérprete un código simple y no ofuscado.
#### Bypass del AMSI

```PowerShell
Set-ExecutionPolicy Bypass -Scope Process -Force
```

## AppLocker

Una lista blanca de aplicaciones de aplicación es una lista de aplicaciones de software aprobadas o ejecutables que se les permite estar presentes y ejecutarse en un sistema. El objetivo es proteger el medio ambiente de malware dañino y software no aprobado que no se alinea con las necesidades específicas de negocio de una organización. [AppLocker](https://docs.microsoft.com/en-us/windows/security/threat-protection/windows-defender-application-control/applocker/what-is-applocker) es la solución de listado de aplicaciones de Microsoft y da a los administradores del sistema control sobre qué aplicaciones y archivos pueden ejecutar los usuarios. Proporciona control granular sobre los ejecutables, scripts, archivos instaladores de Windows, DLLs, aplicaciones empaquetadas y instaladores de aplicaciones en paquete. Es común que las organizaciones bloqueen cmd.exe y PowerShell.exe y escriban acceso a ciertos directorios, pero todo esto se puede eludir. Las organizaciones también se centran a menudo en bloquear la `PowerShell.exe`ejecutables, pero olvídate de los otros [lugares ejecutables de PowerShell](https://www.powershelladmin.com/wiki/PowerShell_Executables_File_System_Locations), como `%SystemRoot%\SysWOW64\WindowsPowerShell\v1.0\powershell.exe`o o `PowerShell_ISE.exe`. Podemos ver que este es el caso en la `AppLocker`normas que se muestran a continuación. Todos los Usuarios de Dominio se les permite ejecutar el ejecutable de PowerShell de 64 bits ubicado en:

`%SystemRoot%\system32\WindowsPowerShell\v1.0\powershell.exe`

Por lo tanto, podemos simplemente llamarlo desde otros lugares. A veces, nos topamos con más estrictos. `AppLocker`políticas que requieren más creatividad para eludir. Estas maneras se abordarán en otros módulos.

#### Usando Get-AppLockerPolicy cmdlet

```PowerShell
Get-AppLockerPolicy -Effective | select -ExpandProperty RuleCollections
```

## LAPS

La [Solución de Contraseña de](https://www.microsoft.com/en-us/download/details.aspx?id=46899) Microsoft [Local Administrator (LAPS)](https://www.microsoft.com/en-us/download/details.aspx?id=46899) se utiliza para aleatorizar y rotar las contraseñas de administrador local en los hosts de Windows y evitar el movimiento lateral. Podemos enumerar qué usuarios de dominios pueden leer el conjunto de contraseña LAPS para máquinas con LAPS instalado y qué máquinas no tienen instalado LAPS. El [LAPSToolkit](https://github.com/leoloobeek/LAPSToolkit) facilita mucho esto con varias funciones. Una es el parlante. `ExtendedRights`para todas las computadoras con LAPS activado. Esto mostrará a los grupos específicamente delegados para leer contraseñas LAPS, que a menudo son usuarios en grupos protegidos. Una cuenta que se ha unido a una computadora a un dominio recibe `All Extended Rights`sobre ese huésped, y este derecho le da a la cuenta la capacidad de leer contraseñas. La enumeración puede mostrar una cuenta de usuario que puede leer la contraseña de LAPS en un host. Esto puede ayudarnos a dirigirse a usuarios específicos de AD que pueden leer contraseñas LAPS.

#### Enumerando controles de seguridad

```PowerShell
Find-LAPSDelegatedGroups
```

El `Find-AdmPwdExtendedRights`comprueba los derechos de cada ordenador con LAPS habilitado para cualquier grupo con acceso leído y usuarios con "Todos los derechos extendidos". Los usuarios con "Todos los Derechos Extendidos" pueden leer contraseñas de LAPS y pueden estar menos protegidos que los usuarios en grupos delegados, por lo que vale la pena comprobarlo.

```powershell
Find-AdmPwdExtendedRights
```

Podemos usar el `Get-LAPSComputers`función para buscar computadoras que tengan activado LAPS cuando expiren las contraseñas, e incluso las contraseñas aleatorizadas en texto claro si nuestro usuario tiene acceso.

```powershell
Get-LAPSComputers
```
