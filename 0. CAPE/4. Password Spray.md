La pulverización de contraseña puede resultar en acceder a los sistemas y potencialmente ganar un punto de apoyo en una red de destino. El ataque implica intentar iniciar sesión en un servicio expuesto usando una contraseña común y una lista más larga de nombres de usuario o direcciones de correo electrónico. Los nombres de usuario y los correos electrónicos pueden haber sido recogidos durante la fase OSINT de la prueba de penetración o nuestros intentos de enumeración inicial.

Una vez conseguida la política de contraseñas, es necesaria una enumeración de usuarios consistente.
Hay varias maneras en las que podemos reunir una lista de usuarios válidos:

- Aprovechando una sesión de SMB NULL para recuperar una lista completa de usuarios de dominios del controlador de dominio
- Utilizando un enlace anónimo LDAP a consultar de forma anónima LDAP y bajar la lista de usuarios de dominios
- Usar una herramienta como `Kerbrute`validar a los usuarios utilizando una lista de palabras de una fuente, como los [nombres de usuario estadísticamente similares](https://github.com/insidetrust/statistically-likely-usernames) GitHub repo, o recopilados usando una herramienta como [el nombre de usuario de linkedin2 para](https://github.com/initstring/linkedin2username) crear una lista de usuarios potencialmente [válidos](https://github.com/insidetrust/statistically-likely-usernames)
- Uso de un conjunto de credenciales de un sistema de ataque Linux o Windows, ya sea proporcionado por nuestro cliente u obtenido a través de otros medios como envenenamiento por la respuesta LLMNR/NBT-NS utilizando `Responder`
