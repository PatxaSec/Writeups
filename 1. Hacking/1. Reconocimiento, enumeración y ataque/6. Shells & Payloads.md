# Shells Jack Us In, Payloads Deliver Us Shells

A es un programa que proporciona a un usuario de computadora una interfaz para ingresar instrucciones en el sistema y ver la salida de texto (Bash, Zsh, cmd y PowerShell, por ejemplo). Como probadores de penetración y profesionales de la seguridad de la información, un shell suele ser el resultado de explotar una vulnerabilidad o eludir las medidas de seguridad para obtener acceso interactivo a un host. Es posible que hayamos escuchado o leído las siguientes frases utilizadas por personas que hablan sobre un compromiso o una sesión de práctica reciente:`shell`

- `"I caught a shell."`
- `"I popped a shell!"`
- `"I dropped into a shell!"`
- `"I'm in!"`

Por lo general, estas frases se traducen en el entendimiento de que esta persona ha explotado con éxito una vulnerabilidad en un sistema y ha podido obtener el control remoto del shell en el sistema operativo de la computadora objetivo. Este es un objetivo común que tendrá un probador de penetración cuando intente acceder a una máquina vulnerable. Observaremos que la mayor parte de este módulo se centrará en lo que viene después de la enumeración e identificación de exploits prometedores.

## Payloads Deliver us Shells

Dentro de la industria de TI en su conjunto, a se puede definir de diferentes maneras:`payload`

- `Networking`: La porción de datos encapsulados de un paquete que atraviesa las redes informáticas modernas.
- `Basic Computing`: Una carga útil es la parte de un conjunto de instrucciones que define la acción que se va a realizar. Se han eliminado los encabezados y la información del protocolo.
- `Programming`: La parte de datos a la que se hace referencia o que transporta la instrucción del lenguaje de programación.
- `Exploitation & Security`: Una carga útil se elabora con la intención de explotar una vulnerabilidad en un sistema informático. El término carga útil puede describir varios tipos de malware, incluido, entre otros, el ransomware.`code`

En este módulo, trabajaremos con muchos tipos diferentes y métodos de entrega dentro del contexto de otorgarnos acceso a un host y establecer sesiones con sistemas vulnerables.`payloads``remote shell`

Somos un probador de penetración que trabaja para CAT5 Security, y nos estamos preparando para realizar un compromiso para nuestro cliente, Inlanefreight. Los miembros más veteranos del equipo de CAT5 quieren ver nuestras habilidades con los shells y las cargas útiles antes de incluirnos en el compromiso en vivo.

La evaluación final de este módulo es una serie de desafíos creados por CAT5 Security para poner a prueba nuestras habilidades. Completar el desafío significa que ha aprobado la evaluación.

#### Conceptos básicos de Shell

- Replicar la posibilidad de obtener un enlace y un shell inverso.
- Enlazar shell en el host de Linux.
- Shell inverso en el host de Windows.

#### Conceptos básicos de la carga útil

- Demuestre el lanzamiento de una carga útil desde MSF.
- Demuestre la búsqueda y creación de una carga útil desde PoC en ExploitDB.
- Demostrar conocimiento de la creación de carga útil.

#### Obtener un shell en Windows

- Usando los resultados de reconocimiento proporcionados, cree o use una carga útil que explote el host y proporcione un shell de vuelta.

#### Obtener un shell en Linux

- Usando los resultados de reconocimiento proporcionados, cree o use una carga útil para explotar el host y establecer una sesión de shell.

#### Aterrizaje de un Web Shell

- Demostrar conocimiento de shells web y aplicaciones web comunes mediante la identificación de una aplicación web común y su lenguaje correspondiente.
