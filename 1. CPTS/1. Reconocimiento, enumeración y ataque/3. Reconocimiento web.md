# CheatSheet

El reconocimiento web es el primer paso en cualquier evaluación de seguridad o compromiso de pruebas de penetración. Es similar a la investigación inicial de un detective, reuniendo meticulosamente pistas y pruebas sobre un objetivo antes de formular un plan de acción. En el ámbito digital, esto se traduce en acumular información sobre un sitio web o una aplicación web para identificar posibles vulnerabilidades, errores de configuración de seguridad y activos valiosos.

Los objetivos principales del reconocimiento web giran en torno a obtener una comprensión integral de la huella digital del objetivo. Esto incluye:

- `Identifying Assets`: La detección de todos los dominios, subdominios y direcciones IP asociados proporciona un mapa de la presencia en línea del objetivo.
- `Uncovering Hidden Information`: El reconocimiento web tiene como objetivo descubrir directorios, archivos y tecnologías que no son evidentes y que podrían servir como puntos de entrada para un atacante.
- `Analyzing the Attack Surface`: Al identificar los puertos abiertos, los servicios en ejecución y las versiones de software, puede evaluar las posibles vulnerabilidades y debilidades del objetivo.
- `Gathering Intelligence`: La recopilación de información sobre los empleados, las direcciones de correo electrónico y las tecnologías utilizadas puede ayudar en los ataques de ingeniería social o identificar vulnerabilidades específicas asociadas con cierto software.

El reconocimiento web se puede llevar a cabo utilizando técnicas activas o pasivas, cada una con sus propias ventajas e inconvenientes:

|Tipo|Descripción|Riesgo de detección|Ejemplos|
|---|---|---|---|
|Reconocimiento activo|Implica interactuar directamente con el sistema de destino, como el envío de sondeos o solicitudes.|Superior|Escaneo de puertos, escaneo de vulnerabilidades, mapeo de red|
|Reconocimiento pasivo|Recopila información sin interactuar directamente con el objetivo, basándose en datos disponibles públicamente.|Bajar|Consultas de motores de búsqueda, búsquedas de WHOIS, enumeración de DNS, análisis de archivos web, redes sociales|

## WHOIS

WHOIS es un protocolo de consulta y respuesta que se utiliza para recuperar información sobre nombres de dominio, direcciones IP y otros recursos de Internet. Es esencialmente un servicio de directorio que detalla quién es el propietario de un dominio, cuándo se registró, información de contacto y más. En el contexto del reconocimiento web, las búsquedas de WHOIS pueden ser una valiosa fuente de información, ya que pueden revelar la identidad del propietario del sitio web, su información de contacto y otros detalles que podrían usarse para investigaciones posteriores o ataques de ingeniería social.

Por ejemplo, si quisieras saber quién es el dueño del dominio, podrías ejecutar el siguiente comando en tu terminal:`example.com`

Código: bash

```bash
whois example.com
```

Esto devolvería una gran cantidad de información, incluido el registrador, las fechas de registro y vencimiento, los servidores de nombres y la información de contacto del propietario del dominio.

Sin embargo, es importante tener en cuenta que los datos de WHOIS pueden ser inexactos o estar ocultos intencionalmente, por lo que siempre es aconsejable verificar la información de múltiples fuentes. Los servicios de privacidad también pueden enmascarar al verdadero propietario de un dominio, lo que dificulta la obtención de información precisa a través de WHOIS.

## DNS

El Sistema de Nombres de Dominio (DNS) funciona como el GPS de Internet, traduciendo los nombres de dominio fáciles de usar en las direcciones IP numéricas que los ordenadores utilizan para comunicarse. Al igual que el GPS convierte el nombre de un destino en coordenadas, el DNS garantiza que su navegador llegue al sitio web correcto al hacer coincidir su nombre con su dirección IP. Esto elimina la memorización de direcciones numéricas complejas, lo que hace que la navegación web sea fluida y eficiente.

El comando le permite consultar servidores DNS directamente, recuperando información específica sobre nombres de dominio. Por ejemplo, si desea encontrar la dirección IP asociada con , puede ejecutar el siguiente comando:`dig``example.com`

Código: bash

```bash
dig example.com A
```

Este comando indica que se consulte el DNS para el registro (que asigna un nombre de host a una dirección IPv4) de . El resultado suele incluir la dirección IP solicitada, junto con detalles adicionales sobre la consulta y la respuesta. Al dominar el comando y comprender los diversos tipos de registros DNS, obtiene la capacidad de extraer información valiosa sobre la infraestructura y la presencia en línea de un objetivo.`dig``A``example.com``dig`

Los servidores DNS almacenan varios tipos de registros, cada uno de los cuales tiene un propósito específico:

|Tipo de registro|Descripción|
|---|---|
|Un|Asigna un nombre de host a una dirección IPv4.|
|AAAA|Asigna un nombre de host a una dirección IPv6.|
|CNAME|Crea un alias para un nombre de host, apuntándolo a otro nombre de host.|
|MX|Especifica los servidores de correo responsables de gestionar el correo electrónico del dominio.|
|NS|Delega una zona DNS a un servidor de nombres autoritativo específico.|
|TXT|Almacena información de texto arbitraria.|
|SOA|Contiene información administrativa sobre una zona DNS.|

## Subdominios

Los subdominios son esencialmente extensiones de un nombre de dominio principal, que a menudo se utilizan para organizar diferentes secciones o servicios dentro de un sitio web. Por ejemplo, una empresa puede usarlo para su servidor de correo electrónico o para su blog.`mail.example.com``blog.example.com`

Desde el punto de vista del reconocimiento, los subdominios son increíblemente valiosos. Pueden exponer superficies de ataque adicionales, revelar servicios ocultos y proporcionar pistas sobre la estructura interna de la red de un objetivo. Los subdominios pueden alojar servidores de desarrollo, entornos de ensayo o incluso aplicaciones olvidadas que no se han protegido correctamente.

El proceso de detección de subdominios se conoce como enumeración de subdominios. Hay dos enfoques principales para la enumeración de subdominios:

|Acercarse|Descripción|Ejemplos|
|---|---|---|
|`Active Enumeration`|Interactúa directamente con los servidores DNS del objetivo o utiliza herramientas para sondear subdominios.|Forzamiento bruto, transferencias de zona DNS|
|`Passive Enumeration`|Recopila información sobre subdominios sin interactuar directamente con el objetivo, basándose en fuentes públicas.|Registros de transparencia de certificados (CT), consultas de motores de búsqueda|

`Active enumeration` puede ser más exhaustivo, pero conlleva un mayor riesgo de detección. Por el contrario, es más sigiloso, pero es posible que no descubra todos los subdominios. La combinación de ambas técnicas puede aumentar significativamente la probabilidad de descubrir una lista completa de subdominios asociados con su objetivo, ampliando su comprensión de su presencia en línea y posibles vulnerabilidades.`passive enumeration`

### Fuerza bruta de subdominios

La fuerza bruta de subdominios es una técnica proactiva utilizada en el reconocimiento web para descubrir subdominios que pueden no ser fácilmente evidentes a través de métodos pasivos. Implica generar sistemáticamente muchos nombres de subdominio potenciales y probarlos con el servidor DNS del objetivo para ver si existen. Este enfoque puede revelar subdominios ocultos que pueden alojar información valiosa, servidores de desarrollo o aplicaciones vulnerables.

Una de las herramientas más versátiles para la fuerza bruta de subdominios es . Esta potente herramienta de línea de comandos combina varias técnicas de enumeración de DNS, incluida la fuerza bruta basada en diccionarios, para descubrir subdominios asociados con su objetivo.`dnsenum`

Para usarlo para la fuerza bruta de subdominios, normalmente le proporcionará el dominio de destino y una lista de palabras que contenga posibles nombres de subdominio. A continuación, la herramienta consultará sistemáticamente el servidor DNS de cada posible subdominio e informará de los que existan.`dnsenum`

Por ejemplo, el siguiente comando intentaría forzar los subdominios mediante una lista de palabras denominada :`example.com``subdomains.txt`

Código: bash

```bash
dnsenum example.com -f subdomains.txt
```

### Traslados de zona

Las transferencias de zona DNS, también conocidas como solicitudes AXFR (Asynchronous Full Transfer), ofrecen una mina de oro potencial de información para el reconocimiento web. Una transferencia de zona es un mecanismo para replicar datos DNS entre servidores. Cuando una transferencia de zona se realiza correctamente, proporciona una copia completa del archivo de zona DNS, que contiene una gran cantidad de detalles sobre el dominio de destino.

Este archivo de zona enumera todos los subdominios del dominio, sus direcciones IP asociadas, configuraciones del servidor de correo y otros registros DNS. Esto es similar a obtener un plano de la infraestructura DNS del objetivo para un experto en reconocimiento.

Para intentar una transferencia de zona, puede utilizar el comando con la opción (transferencia de zona completa). Por ejemplo, para solicitar una transferencia de zona desde el servidor DNS para el dominio, ejecutaría:`dig``axfr``ns1.example.com``example.com`

Código: bash

```bash
dig @ns1.example.com example.com axfr
```

Sin embargo, no siempre se permiten los transbordos de zona. Muchos servidores DNS están configurados para restringir las transferencias de zona solo a servidores secundarios autorizados. Sin embargo, los servidores mal configurados pueden permitir transferencias de zona desde cualquier fuente, exponiendo inadvertidamente información confidencial.

### Anfitriones virtuales

El alojamiento virtual es una técnica que permite que varios sitios web compartan una sola dirección IP. Cada sitio web está asociado a un nombre de host único, que se utiliza para dirigir las solicitudes entrantes al sitio correcto. Esta puede ser una forma rentable para que las organizaciones alojen varios sitios web en un solo servidor, pero también puede crear un desafío para el reconocimiento web.

Dado que varios sitios web comparten la misma dirección IP, el simple escaneo de la IP no revelará todos los sitios alojados. Necesita una herramienta que pueda probar diferentes nombres de host con la dirección IP para ver cuáles responden.

Gobuster es una herramienta versátil que se puede utilizar para varios tipos de fuerza bruta, incluido el descubrimiento de host virtual. Su modo está diseñado para enumerar hosts virtuales mediante el envío de solicitudes a la dirección IP de destino con diferentes nombres de host. Si un host virtual está configurado para un nombre de host específico, Gobuster recibirá una respuesta del servidor web.`vhost`

Para usar Gobuster para forzar hosts virtuales, necesitará una lista de palabras que contenga posibles nombres de host. A continuación, se muestra un comando de ejemplo:

Código: bash

```bash
gobuster vhost -u http://192.0.2.1 -w hostnames.txt
```

En este ejemplo, especifica la dirección IP de destino y especifica el archivo de lista de palabras. A continuación, Gobuster probará sistemáticamente cada nombre de host de la lista de palabras e informará de cualquiera que dé lugar a una respuesta válida del servidor web.`-u``-w`

### Registros de transparencia de certificados (CT)

Los registros de Certificate Transparency (CT) ofrecen un tesoro de información de subdominios para el reconocimiento pasivo. Estos registros de acceso público registran los certificados SSL/TLS emitidos para los dominios y sus subdominios, lo que sirve como medida de seguridad para evitar certificados fraudulentos. Para el reconocimiento, ofrecen una ventana a subdominios potencialmente pasados por alto.

El sitio web proporciona una interfaz de búsqueda para los registros de TC. Para extraer subdominios de manera eficiente usando dentro de su terminal, puede usar un comando como este:`crt.sh``crt.sh`

Código: bash

```bash
curl -s "https://crt.sh/?q=%25.example.com&output=json" | jq -r '.[].name_value' | sed 's/\*\.//g' | sort -u
```

Este comando obtiene datos con formato JSON de for (el es un comodín), extrae los nombres de dominio mediante , elimina los prefijos comodín () con , y, por último, ordena y deduplica los resultados.`crt.sh``example.com``%``jq``*.``sed`

## Rastreo web

El rastreo web es la exploración automatizada de la estructura de un sitio web. Un rastreador web, o araña, navega sistemáticamente a través de las páginas web siguiendo enlaces, imitando el comportamiento de navegación de un usuario. Este proceso traza la arquitectura del sitio y recopila información valiosa incrustada en las páginas.

Un archivo crucial que guía a los rastreadores web es . Este archivo reside en el directorio raíz de un sitio web y dicta qué áreas están fuera del alcance de los rastreadores. El análisis puede revelar directorios ocultos o áreas sensibles que el propietario del sitio web no desea que sean indexadas por los motores de búsqueda.`robots.txt``robots.txt`

`Scrapy` es un marco de Python potente y eficiente para proyectos de rastreo y raspado web a gran escala. Proporciona un enfoque estructurado para definir reglas de rastreo, extraer datos y controlar varios formatos de salida.

Aquí hay un ejemplo básico de araña Scrapy para extraer enlaces:`example.com`

Código: python

```python
import scrapy

class ExampleSpider(scrapy.Spider):
    name = "example"
    start_urls = ['http://example.com/']

    def parse(self, response):
        for link in response.css('a::attr(href)').getall():
            if any(link.endswith(ext) for ext in self.interesting_extensions):
                yield {"file": link}
            elif not link.startswith("#") and not link.startswith("mailto:"):
                yield response.follow(link, callback=self.parse)
```

Después de ejecutar la araña Scrapy, tendrá un archivo que contiene datos raspados (por ejemplo, ). Puede analizar estos resultados mediante herramientas de línea de comandos estándar. Por ejemplo, para extraer todos los enlaces:`example_data.json`

Código: bash

```bash
jq -r '.[] | select(.file != null) | .file' example_data.json | sort -u
```

Este comando se utiliza para extraer vínculos, aislar extensiones de archivo, ordenarlas y contar sus apariciones. Al examinar los datos extraídos, puede identificar patrones, anomalías o archivos confidenciales que podrían ser de interés para una investigación más profunda.`jq``awk``sort``uniq -c`

## Descubrimiento de motores de búsqueda

Aprovechar los motores de búsqueda para el reconocimiento implica utilizar sus vastos índices de contenido web para descubrir información sobre su objetivo. Esta técnica pasiva, a menudo denominada recopilación de inteligencia de código abierto (OSINT), puede proporcionar información valiosa sin interactuar directamente con los sistemas del objetivo.

Mediante el empleo de operadores de búsqueda avanzada y consultas especializadas conocidas como "Google Dorks", puede identificar información específica oculta en los resultados de búsqueda. A continuación, se muestra una tabla de algunos operadores de búsqueda útiles para el reconocimiento web:

|Operador|Descripción|Ejemplo|
|---|---|---|
|`site:`|Restringe los resultados de búsqueda a un sitio web específico.|`site:example.com "password reset"`|
|`inurl:`|Busca un término específico en la URL de una página.|`inurl:admin login`|
|`filetype:`|Limita los resultados a archivos de un tipo específico.|`filetype:pdf "confidential report"`|
|`intitle:`|Busca un término dentro del título de una página.|`intitle:"index of" /backup`|
|`cache:`|Muestra la versión almacenada en caché de una página web.|`cache:example.com`|
|`"search term"`|Busca la frase exacta entre comillas.|`"internal error" site:example.com`|
|`OR`|Combina varios términos de búsqueda.|`inurl:admin OR inurl:login`|
|`-`|Excluye términos específicos de los resultados de búsqueda.|`inurl:admin -intext:wordpress`|

Al combinar de manera creativa estos operadores y elaborar consultas específicas, puede descubrir documentos confidenciales, directorios expuestos, páginas de inicio de sesión y otra información valiosa que puede ayudar en sus esfuerzos de reconocimiento.

## Archivos Web

Los archivos web son repositorios digitales que almacenan instantáneas de sitios web a lo largo del tiempo, proporcionando un registro histórico de su evolución. Entre estos archivos, Wayback Machine es el recurso más completo y accesible para el reconocimiento web.

Wayback Machine, un proyecto de Internet Archive, ha estado archivando la web durante más de dos décadas, capturando miles de millones de páginas web de todo el mundo. Esta recopilación masiva de datos históricos puede ser un recurso invaluable para los investigadores de seguridad.

|Característica|Descripción|Caso de uso en reconocimiento|
|---|---|---|
|`Historical Snapshots`|Ver versiones anteriores de sitios web, incluidas las páginas, el contenido y los cambios de diseño.|Identifique el contenido o la funcionalidad del sitio web anterior que ya no está disponible.|
|`Hidden Directories`|Explore los directorios y archivos que pueden haber sido eliminados u ocultos de la versión actual del sitio web.|Descubra información confidencial o copias de seguridad que se dejaron accesibles inadvertidamente en versiones anteriores.|
|`Content Changes`|Realice un seguimiento de los cambios en el contenido del sitio web, incluidos el texto, las imágenes y los enlaces.|Identifique patrones en las actualizaciones de contenido y evalúe la evolución de la postura de seguridad de un sitio web.|

Al aprovechar Wayback Machine, puede obtener una perspectiva histórica de la presencia en línea de su objetivo, lo que puede revelar vulnerabilidades que pueden haberse pasado por alto en la versión actual del sitio web.

## Tipos de reconocimiento

El reconocimiento web abarca dos metodologías fundamentales: el reconocimiento y el reconocimiento. Cada enfoque ofrece ventajas y desafíos distintos, y comprender sus diferencias es crucial para una recopilación adecuada de información.`active` `passive`

### Reconocimiento activo

En el reconocimiento activo, el atacante para recopilar información. Esta interacción puede adoptar varias formas:`directly interacts with the target system`

|Técnica|Descripción|Ejemplo|Herramientas|Riesgo de detección|
|---|---|---|---|---|
|`Port Scanning`|Identificación de puertos abiertos y servicios que se ejecutan en el destino.|Uso de Nmap para escanear un servidor web en busca de puertos abiertos como 80 (HTTP) y 443 (HTTPS).|Nmap, Masscan, Unicornscan|Alto: La interacción directa con el objetivo puede activar sistemas de detección de intrusos (IDS) y firewalls.|
|`Vulnerability Scanning`|Sondear el objetivo en busca de vulnerabilidades conocidas, como software obsoleto o configuraciones incorrectas.|Ejecución de Nessus en una aplicación web para comprobar si hay fallos de inyección SQL o vulnerabilidades de secuencias de comandos entre sitios (XSS).|Nessus, OpenVAS, Nikto|Alto: los escáneres de vulnerabilidades envían cargas útiles de exploits que las soluciones de seguridad pueden detectar.|
|`Network Mapping`|Mapeo de la topología de red del destino, incluidos los dispositivos conectados y sus relaciones.|Uso de traceroute para determinar la ruta que tardan los paquetes en llegar al servidor de destino, lo que revela posibles saltos de red e infraestructura.|Traceroute, Nmap|Medio a alto: El tráfico de red excesivo o inusual puede levantar sospechas.|
|`Banner Grabbing`|Recuperación de información de banners mostrados por los servicios que se ejecutan en el destino.|Conectarse a un servidor web en el puerto 80 y examinar el banner HTTP para identificar el software y la versión del servidor web.|Netcat, rizo|Bajo: El acaparamiento de banners suele implicar una interacción mínima, pero aún se puede registrar.|
|`OS Fingerprinting`|Identificar el sistema operativo que se ejecuta en el destino.|Uso de las capacidades de detección del sistema operativo de Nmap () para determinar si el objetivo está ejecutando Windows, Linux u otro sistema operativo.`-O`|Nmap, Xprobe2|Bajo: La huella digital del sistema operativo suele ser pasiva, pero se pueden detectar algunas técnicas avanzadas.|
|`Service Enumeration`|Determinar las versiones específicas de los servicios que se ejecutan en puertos abiertos.|Uso de la detección de versiones de servicio de Nmap () para determinar si un servidor web está ejecutando Apache 2.4.50 o Nginx 1.18.0.`-sV`|Nmapa|Bajo: de forma similar a la captura de banners, la enumeración de servicios se puede registrar, pero es menos probable que desencadene alertas.|
|`Web Spidering`|Rastreo del sitio web de destino para identificar páginas web, directorios y archivos.|Ejecutar un rastreador web como Burp Suite Spider u OWASP ZAP Spider para trazar la estructura de un sitio web y descubrir recursos ocultos.|Burp Suite Spider, OWASP ZAP Spider, Scrapy (personalizable)|De bajo a medio: se puede detectar si el comportamiento del rastreador no está configurado cuidadosamente para imitar el tráfico legítimo.|

El reconocimiento activo proporciona una visión directa y, a menudo, más completa de la infraestructura y la postura de seguridad del objetivo. Sin embargo, también conlleva un mayor riesgo de detección, ya que las interacciones con el objetivo pueden desencadenar alertas o levantar sospechas.

### Reconocimiento pasivo

Por el contrario, el reconocimiento pasivo consiste en recopilar información sobre el objetivo que lo acompaña. Esto se basa en el análisis de la información y los recursos disponibles públicamente, tales como:`without directly interacting`

|Técnica|Descripción|Ejemplo|Herramientas|Riesgo de detección|
|---|---|---|---|---|
|`Search Engine Queries`|Utilizar los motores de búsqueda para descubrir información sobre el objetivo, incluidos sitios web, perfiles de redes sociales y artículos de noticias.|Buscar en Google "" para encontrar información de los empleados o perfiles de redes sociales.`[Target Name] employees`|Google, DuckDuckGo, Bing y motores de búsqueda especializados (por ejemplo, Shodan)|Muy bajo: Las consultas de los motores de búsqueda son una actividad normal en Internet y es poco probable que activen alertas.|
|`WHOIS Lookups`|Consultar bases de datos WHOIS para recuperar detalles de registro de dominio.|Realizar una búsqueda de WHOIS en un dominio de destino para encontrar el nombre, la información de contacto y los servidores de nombres del registrante.|herramienta de línea de comandos de whois, servicios de búsqueda de WHOIS en línea|Muy bajo: Las consultas de WHOIS son legítimas y no levantan sospechas.|
|`DNS`|Análisis de registros DNS para identificar subdominios, servidores de correo y otras infraestructuras.|Se utiliza para enumerar subdominios de un dominio de destino.`dig`|dig, nslookup, host, dnsenum, feroz, dnsrecon|Muy bajo: Las consultas DNS son esenciales para la navegación por Internet y, por lo general, no se marcan como sospechosas.|
|`Web Archive Analysis`|Examinar instantáneas históricas del sitio web del objetivo para identificar cambios, vulnerabilidades o información oculta.|Usar Wayback Machine para ver versiones anteriores de un sitio web de destino y ver cómo ha cambiado con el tiempo.|Máquina Wayback|Muy bajo: Acceder a las versiones archivadas de los sitios web es una actividad normal.|
|`Social Media Analysis`|Recopilación de información de plataformas de redes sociales como LinkedIn, Twitter o Facebook.|Buscar en LinkedIn a los empleados de una organización objetivo para conocer sus funciones, responsabilidades y posibles objetivos de ingeniería social.|LinkedIn, Twitter, Facebook, herramientas OSINT especializadas|Muy bajo: El acceso a perfiles públicos de redes sociales no se considera intrusivo.|
|`Code Repositories`|Análisis de repositorios de código de acceso público, como GitHub, en busca de credenciales expuestas o vulnerabilidades.|Buscar en GitHub fragmentos de código o repositorios relacionados con el destino que puedan contener información confidencial o vulnerabilidades de código.|GitHub, GitLab|Muy bajo: Los repositorios de código están pensados para el acceso público, y buscarlos no es sospechoso.|

El reconocimiento pasivo generalmente se considera más sigiloso y menos probable que active alarmas que el reconocimiento activo. Sin embargo, puede producir información menos completa, ya que se basa en lo que ya es accesible públicamente.
