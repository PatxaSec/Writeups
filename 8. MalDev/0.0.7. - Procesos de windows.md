
Un **proceso** es un programa en ejecución, iniciado por un usuario o por el sistema.
Consume recursos: memoria, disco y CPU.

---

### **Hilos (Threads)**

- Un proceso puede tener uno o más hilos ejecutándose simultáneamente.
    
- Cada hilo ejecuta instrucciones de manera independiente y puede compartir datos con otros hilos del mismo proceso.
    
- El sistema operativo administra la ejecución de los hilos dentro del proceso.
    

---

### **Memoria de los procesos**

- Los procesos usan **memoria** para datos e instrucciones.
    
- El SO gestiona memoria física y virtual:
    
    - **Memoria virtual:** permite usar más memoria de la físicamente disponible mediante direcciones virtuales divididas en “páginas”.
        

**Tipos de memoria:**

1. **Privada:** exclusiva del proceso.
    
2. **Mapeada:** compartida entre procesos (ej. librerías o archivos compartidos).
    
3. **Image memory:** contiene código y datos del ejecutable, incluyendo DLLs cargadas.
    

---

### **Process Environment Block (PEB)**

- Estructura de datos que contiene información del proceso: parámetros, DLLs cargadas, heap, PID, ruta del ejecutable, etc.
```c
    typedef struct _PEB {
  BYTE                          Reserved1[2];
  BYTE                          BeingDebugged;
  BYTE                          Reserved2[1];
  PVOID                         Reserved3[2];
  PPEB_LDR_DATA                 Ldr;
  PRTL_USER_PROCESS_PARAMETERS  ProcessParameters;
  PVOID                         Reserved4[3];
  PVOID                         AtlThunkSListPtr;
  PVOID                         Reserved5;
  ULONG                         Reserved6;
  PVOID                         Reserved7;
  ULONG                         Reserved8;
  ULONG                         AtlThunkSListPtr32;
  PVOID                         Reserved9[45];
  BYTE                          Reserved10[96];
  PPS_POST_PROCESS_INIT_ROUTINE PostProcessInitRoutine;
  BYTE                          Reserved11[128];
  PVOID                         Reserved12[1];
  ULONG                         SessionId;
} PEB, *PPEB;
```
    
- **Campos importantes:**

- **BeingDebugged:** indica si el proceso está siendo depurado.
- **Ldr:** lista de DLLs cargadas en memoria.

```c
typedef struct _PEB_LDR_DATA {
  BYTE       Reserved1[8];
  PVOID      Reserved2[3];
  LIST_ENTRY InMemoryOrderModuleList;
} PEB_LDR_DATA, *PPEB_LDR_DATA;
```

-  **ProcessParameters:** parámetros de línea de comando y ruta del ejecutable.

```c
typedef struct _RTL_USER_PROCESS_PARAMETERS {
  BYTE           Reserved1[16];
  PVOID          Reserved2[10];
  UNICODE_STRING ImagePathName;
  UNICODE_STRING CommandLine;
} RTL_USER_PROCESS_PARAMETERS, *PRTL_USER_PROCESS_PARAMETERS;
```

- **AtlThunkSListPtr / AtlThunkSListPtr32:** punteros a funciones thunking de DLLs.
        
- **PostProcessInitRoutine:** función ejecutada después de inicializar TLS.
        
- **SessionId:** identifica la sesión de usuario.
        

---

### **Thread Environment Block (TEB)**

- Estructura que contiene información específica de cada hilo: contexto, seguridad, TLS, puntero al PEB del proceso.
    
- **Campos importantes:**
    
    - **ProcessEnvironmentBlock (PEB):** referencia al PEB del proceso.
        
    - **TlsSlots / TlsExpansionSlots:** almacenamiento de datos específicos del hilo.
        
```c
typedef struct _TEB {
  PVOID Reserved1[12];
  PPEB  ProcessEnvironmentBlock;
  PVOID Reserved2[399];
  BYTE  Reserved3[1952];
  PVOID TlsSlots[64];
  BYTE  Reserved4[8];
  PVOID Reserved5[26];
  PVOID ReservedForOle;
  PVOID Reserved6[4];
  PVOID TlsExpansionSlots;
} TEB, *PTEB;
```

---

### **Identificadores y handles**

- Cada proceso tiene un **PID** único, y cada hilo un **TID** único.
    
- Se pueden abrir **handles** para procesos e hilos mediante WinAPI:
    
    - [OpenProcess](https://learn.microsoft.com/en-us/windows/win32/api/processthreadsapi/nf-processthreadsapi-openprocess) → abrir handle de proceso.
        
    - [OpenThread](https://learn.microsoft.com/en-us/windows/win32/api/processthreadsapi/nf-processthreadsapi-openthread) → abrir handle de hilo.
        
    - [CloseHandle](https://learn.microsoft.com/en-us/windows/win32/api/handleapi/nf-handleapi-closehandle) → cerrar handle y evitar fugas de recursos.
