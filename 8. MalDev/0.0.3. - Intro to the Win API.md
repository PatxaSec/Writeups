

La **Windows API** permite que las aplicaciones interactúen con Windows: mostrar información en pantalla, modificar archivos o consultar el registro. Está bien documentada por Microsoft.

---

## **Tipos de datos en Windows API**

### Tipos básicos:

```c
DWORD dwVariable = 42;             // Entero sin signo de 32 bits
SIZE_T sVariable = sizeof(int);    // Tamaño de un objeto (32/64 bits según arquitectura)
void* pVariable = NULL;            // Indica ausencia de tipo (VOID / PVOID)
PVOID pVariable = &SomeData;       // Puntero genérico
HANDLE hFile = CreateFile(...);    // Identifica un objeto gestionado por el SO
HMODULE hModule = GetModuleHandle(...); // Puntero a la base de un módulo
```

### Cadenas:

```c
// ANSI
LPCSTR  lpcString = "Hello, world!"; // constante
PCSTR   pcString  = "Hello, world!"; // constante
LPSTR   lpString  = "Hello, world!"; // modificable
PSTR    pString   = "Hello, world!"; // modificable

// Unicode
LPCWSTR lpwcString = L"Hello, world!"; // constante
PCWSTR  pcwString  = L"Hello, world!"; // constante
LPWSTR  lpwString  = L"Hello, world!"; // modificable
PWSTR   pwString   = L"Hello, world!"; // modificable

wchar_t     wChar    = L'A';
wchar_t*    wcString = L"Hello, world!";
```

### Punteros y aritmética con punteros:

```c
PVOID Pointer = malloc(100);
// Pointer = Pointer + 10; // no permitido
Pointer = (ULONG_PTR)Pointer + 10; // permitido
```

### Equivalencias con punteros:

```c
PHANDLE h = &hFile;  // PHANDLE = HANDLE*
PSIZE_T ps = &sVariable; // PSIZE_T = SIZE_T*
PDWORD pd = &dwVariable; // PDWORD = DWORD*
```

## **Funciones ANSI y Unicode**

- `CreateFileA` → usa cadenas ANSI (LPCSTR)
    
- `CreateFileW` → usa cadenas Unicode (LPCWSTR)
    

```c
char str1[]  = "maldev";  // 7 bytes (incluye null)
wchar_t str2[] = L"maldev"; // 14 bytes, 2 bytes por carácter
```

---

## **Parámetros IN y OUT**

- **IN:** entrada a la función
    
- **OUT:** salida de la función (usando punteros)
    

```c
BOOL HackTheWorld(OUT int* num){
    *num = 123;  // modifica el valor de salida
    return TRUE;
}

int main(){
    int a = 0;
    HackTheWorld(&a); // a = 123
}
```

## **Ejemplo práctico: CreateFileW**

```c
HANDLE hFile = INVALID_HANDLE_VALUE; 
LPCWSTR filePath = L"C:\\Users\\maldevacademy\\Desktop\\maldev.txt";

hFile = CreateFileW(
    filePath,             // Nombre del archivo
    GENERIC_ALL,          // Acceso deseado
    0,                    // Compartir
    NULL,                 // Seguridad
    CREATE_ALWAYS,        // Creación
    FILE_ATTRIBUTE_NORMAL,// Atributos
    NULL                  // Archivo plantilla
);

if (hFile == INVALID_HANDLE_VALUE){
    printf("[-] CreateFileW Api Function Failed With Error : %d\n", GetLastError());
    return -1;
}
```

## **Depuración de errores de Windows API**

- Funciones que fallan retornan valores no descriptivos (`INVALID_HANDLE_VALUE`).
    
- Para conocer la causa, se usa `GetLastError()`.
    

Códigos comunes:

```text
5   -> ERROR_ACCESS_DENIED
2   -> ERROR_FILE_NOT_FOUND
87  -> ERROR_INVALID_PARAMETER
```

---

## **Depuración de Windows Native API (NTAPI)**

- Funciones exportadas desde `ntdll.dll` retornan **NTSTATUS** en lugar de `GetLastError()`.
    
- `STATUS_SUCCESS = 0`, otros valores indican fallo.
    

```c
NTSTATUS STATUS = NativeSyscallExample(...);
if (STATUS != STATUS_SUCCESS){
    printf("[!] NativeSyscallExample Failed With Status : 0x%0.8X \n", STATUS);
}

// Usando macro NT_SUCCESS
#define NT_SUCCESS(Status) (((NTSTATUS)(Status)) >= 0)

if (!NT_SUCCESS(STATUS)){
    printf("[!] NativeSyscallExample Failed With Status : 0x%0.8X \n", STATUS);
}
```
